How to Connect

1
Choose how to add your server

On iPhone or iPad, the QuickTUI Setup Wizard supports three paths:

  • Quick Setup — SSH into your server and install QuickTUI automatically.
  • Scan QR Code — run quicktui-server --qrcode on an existing server, then scan it.
  • Add Existing Server — enter the server URL and access token manually.
In a browser, there is no setup wizard. Open the server URL directly and enter the token when prompted.
2
Install and start the server

If you use Quick Setup, the app will SSH into your Mac or Linux server, run the installer, and configure QuickTUI for you.

If you install it yourself, run:

curl -fsSL https://quicktui.ai/q.sh | sh

The installer writes ~/.config/quicktui/config, then asks whether to register a background service. The default choice is Yes. The default listen address is 0.0.0.0:8022.

If you skip service registration, run the exact command printed by the installer. Manual startup uses flags and environment variables directly; it does not automatically load ~/.config/quicktui/config.
3
Get your address and token

If you are adding an existing server manually, you need the server URL and access token.

To find your server's local IP:

macOS:  ipconfig getifaddr en0 || ipconfig getifaddr en1
Linux:  hostname -I | awk '{print $1}'

If you are using the background service, you can view the saved token and address with:

cat ~/.config/quicktui/config

For QR onboarding, just run quicktui-server --qrcode on the server and scan it in the app.

4
Open the app or browser

On iPhone or iPad, open the QuickTUI app and choose the setup path that matches your server:

  • Quick Setup if the server is not installed yet and SSH access is available.
  • Scan QR Code if you ran quicktui-server --qrcode.
  • Add Existing Server if you already have the server URL and token.

In a browser, navigate to your server URL, for example http://192.168.1.100:8022, then enter the token on first visit.

5
Connect and manage sessions

Tap Connect. You will see your tmux sessions. Select an existing one or create a new session with the New Session button.

Connecting over the internet? Use Tailscale, WireGuard, or Cloudflare Tunnel to expose your server safely.

FAQ

Installation

Checksum verification failed during install

This usually means the download was interrupted or corrupted. Try running the installer again. If the problem persists, the release assets may have been updated — wait a few minutes and retry.

You can also verify your network connection and ensure nothing is intercepting HTTPS traffic (e.g. a corporate proxy).

tmux is not installed or the version is too old

QuickTUI requires tmux 3.2 or later. The installer can usually install tmux for you. If you want to install or upgrade it yourself:

macOS:   brew install tmux
Ubuntu:  sudo apt install tmux
CentOS:  sudo yum install tmux

If the installer falls back to a prebuilt tmux archive, it will save QUICKTUI_TMUX_BIN in ~/.config/quicktui/config so the background service can keep finding that exact binary.

Server & Service

What can I configure in the config file?

If you use the background service, its source of truth is ~/.config/quicktui/config. The installer writes this file even if you skip service registration, but a manual quicktui-server process does not read the config file automatically on normal startup.

# Access token — required
QUICKTUI_TOKEN=your-secret-token

# Listen address and port (default: 0.0.0.0:8022)
QUICKTUI_ADDR=0.0.0.0:8022

# TERM value passed to tmux (default: xterm-256color)
QUICKTUI_TERM=xterm-256color

# Locale passed to tmux (default: en_US.UTF-8)
QUICKTUI_LANG=en_US.UTF-8

# Optional: absolute path to tmux, used when the installer bootstrapped tmux
QUICKTUI_TMUX_BIN=/Users/you/.local/tmux/tmux

After editing the config file, restart the service:

launchctl kickstart -k gui/$(id -u)/ai.quicktui   # macOS
systemctl --user restart quicktui                  # Linux

If you run quicktui-server manually, pass flags or env vars directly instead:

QUICKTUI_TOKEN=your-secret-token \
QUICKTUI_ADDR=0.0.0.0:8022 \
~/.local/bin/quicktui-server
QuickTUI says "token is required"

There are two common cases:

  • Background service: make sure ~/.config/quicktui/config exists and contains QUICKTUI_TOKEN=....
  • Manual start: pass --token or set QUICKTUI_TOKEN in the environment. Normal server startup does not read the config file automatically.
Service was registered but did not start successfully

The installer now checks whether the service actually becomes reachable after registration. If this step fails, try starting it manually first:

launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/ai.quicktui.plist   # macOS
systemctl --user start quicktui                                             # Linux

Then check the logs. Common causes are a missing QUICKTUI_TOKEN, an invalid QUICKTUI_TMUX_BIN, or a QUICKTUI_TERM value that is not available on the server.

Can't connect — connection refused

First, verify that the server is running:

launchctl print gui/$(id -u)/ai.quicktui   # macOS service
systemctl --user status quicktui           # Linux service

If you skipped service registration, start the server with the same token and address you intend to use, for example:

QUICKTUI_TOKEN=your-secret-token \
QUICKTUI_ADDR=0.0.0.0:8022 \
~/.local/bin/quicktui-server

If the process is running, make sure your firewall or tunnel allows the port from QUICKTUI_ADDR.

Terminal rendering is broken when using Ghostty

Ghostty uses TERM=xterm-ghostty. If the server or service should use that terminfo entry too, set:

Edit ~/.config/quicktui/config and set:

QUICKTUI_TERM=xterm-ghostty

Then restart the service, or rerun your manual server command with --term xterm-ghostty.

PTY spawn failed / internal server error when opening a session

This usually means the server could not launch or attach tmux. Check the logs first:

tail -f ~/Library/Logs/QuickTUI/stdout.log ~/Library/Logs/QuickTUI/stderr.log   # macOS service
journalctl --user -u quicktui -f             # Linux service

If the config contains QUICKTUI_TMUX_BIN, it must be an absolute path to a real tmux binary. If the installer added that line for a bundled tmux, leave it intact unless the binary moved.

Also verify that QUICKTUI_TERM and QUICKTUI_LANG are valid for the environment where the server is running.

How do I view server logs?

If QuickTUI is running as a background service:

# macOS
tail -f ~/Library/Logs/QuickTUI/stdout.log ~/Library/Logs/QuickTUI/stderr.log

# Linux
journalctl --user -u quicktui -f

If you started quicktui-server manually, view the output in the terminal where you launched it.

How do I change the listen address or port?

If you use the background service, edit ~/.config/quicktui/config and update QUICKTUI_ADDR:

QUICKTUI_ADDR=0.0.0.0:9000
launchctl kickstart -k gui/$(id -u)/ai.quicktui   # macOS
systemctl --user restart quicktui                  # Linux

If you run the server manually, pass the address directly on startup:

~/.local/bin/quicktui-server \
  --token your-secret-token \
  --addr 0.0.0.0:9000

You can also set a different address and port at install time. The installer will save the final value to QUICKTUI_ADDR for service-based setups:

curl -fsSL https://quicktui.ai/q.sh | sh -s -- --addr 127.0.0.1 --port 9000
How do I uninstall QuickTUI? 
curl -fsSL https://quicktui.ai/q.sh | sh -s -- --uninstall

This stops the service when possible and removes the binary, config, known launchd/systemd service files, and logs. It also cleans up leftover service files from partial installs where the binary is already gone.

tmux

My session list is empty

QuickTUI shows your current tmux sessions. If none exist yet, create one from the app or browser with New Session, or create one on the server yourself:

tmux new-session -s mysession

iOS App & Browser

What is the access token?

The access token is a secret you set during installation. It protects your server from unauthorized access. You can find it in ~/.config/quicktui/config on your server:

cat ~/.config/quicktui/config

Enter this token when prompted on first login in the app or browser.

Can I connect over the internet?

Yes. You need your server to be reachable from the internet — either via a public IP, a VPN, or a tunnel service such as Tailscale or Cloudflare Tunnel. Make sure the configured port (8022 by default) is accessible and your token is kept secret.

The app or browser shows "Session not found"

This comes directly from the server. It can mean the tmux session was closed or renamed, but it can also mean the tmux server is no longer running or cannot be reached.

Refresh the session list, then open an existing session or create a new one.
